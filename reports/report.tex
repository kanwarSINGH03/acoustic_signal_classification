\begin{abstract}
Miners traditionally strike the roof with a scaling bar and judge safety by the impact sound (\emph{``Drummy''} vs \emph{``Tight''}). This judgment is subjective and can be difficult in complex geometries. We present a two-phase project to automatically classify impact audio. Phase~1 used studio-microphone data and PCA+ML baselines; Phase~2 introduced a custom device mounted on the scaling bar with Bluetooth connectivity, and evaluated end-to-end waveform CNNs and MFCC-based deep models. Results show that deep models, especially MFCC+ResNet, markedly outperform classical baselines. The Phase~1 ResNet saturates near 98\% accuracy; Phase~2 MFCC+ResNet achieves 93\% with balanced class recall. We discuss design choices (tanh activations, label smoothing), dataset issues (mishits), and future directions (unified variable-length front-ends, automated mishit filtering, embedded deployment).
\end{abstract}

\section{Introduction and Project Overview}
Scaling bars are steel or aluminum rods (2--3\,m) used to strike the mine roof before work begins. Workers listen for a \emph{Drummy} (loose) or \emph{Tight} (safe) sound. This process depends on experience and is error-prone in challenging conditions (intersecting rooms, high roofs). Our objective is to learn a mapping from impact audio to \{Drummy, Tight\} labels to reduce subjectivity.

\paragraph{Phase 2 Device.}
A custom device was designed and mounted on the scaling bar to:
\begin{itemize}
  \item detect impacts and record audio,
  \item allow optional in-field labeling,
  \item transmit audio via Bluetooth to an app,
  \item receive classifier outputs and present feedback via RGB LED.
\end{itemize}

\section{Deep Learning and Generalization Techniques}
PCA features are global and do not capture transient temporal dynamics. We therefore tested end-to-end CNNs with skip connections i.e. custom ResNets on raw waveforms and hybrids using MFCCs. Unless noted, the split is \textbf{90/10} (train/test) since we do not have much data to perform a smaller train split.
Train/Val routine can be viewed in \textbf{/evaluation/report\_experiments.ipynb}.
Generalization techniques included:
\begin{itemize}
  \item \textbf{tanh} after convolutional stages,
  \item \textbf{BatchNorm1d} in between the two Residual Blocks in Phase~2 ResNet,
  \item \textbf{Dropout} \(=0.6\) in classifier heads,
  \item \textbf{Label smoothing} \(\epsilon=0.1\).
\end{itemize}

\subsection{Label Smoothing in Binary Classification}
Label smoothing replaces hard one-hot targets with softened targets to reduce overconfidence and improve generalization. For $K$ classes and smoothing parameter $\varepsilon\in[0,1)$, the smoothed target for class $i$ is
\begin{equation}
\label{eq:ls-general}
q_i \;=\; (1-\varepsilon)\,y_i \;+\; \frac{\varepsilon}{K},
\end{equation}
where $y_i\in\{0,1\}$ is the one-hot target.

\paragraph{Binary case ($K{=}2$).}
Let the positive class be indexed as $1$ and let $p\in(0,1)$ be the model's predicted probability for that class. With label smoothing,
\[
y_{\text{smooth}} \;=\; (1-\varepsilon)\cdot y \;+\; \frac{\varepsilon}{2},
\]
so the target vector becomes
\[
y=1 \;\Rightarrow\; [\,1{-}\varepsilon,\;\varepsilon\,], 
\qquad
y=0 \;\Rightarrow\; [\,\varepsilon,\;1{-}\varepsilon\,].
\]

\paragraph{Label smoothing with CE Loss.}
\textbf{Cross-entropy (CE)} for a $K$-class classifier with predicted probabilities $\mathbf{p}$ and one-hot target $\mathbf{y}$ is
\[
\mathcal{L}_{\text{CE}}(\mathbf{p},\mathbf{y}) \;=\; -\sum_{k=1}^{K} y_k \,\ln p_k .
\]
With \textbf{label smoothing} ($\varepsilon$), the target becomes
\[
\mathbf{q} \;=\; (1-\varepsilon)\,\mathbf{y} \;+\; \frac{\varepsilon}{K}\,\mathbf{1},
\]
and the loss is
\[
\mathcal{L}_{\text{CE+LS}}(\mathbf{p},\mathbf{q}) \;=\; -\sum_{k=1}^{K} q_k \,\ln p_k .
\]
For \textbf{binary} classification ($K{=}2$) with $y{=}1$ and $\varepsilon{=}0.1$,
\[
\mathbf{q} \;=\; [\,0.05,\;0.95\,], \quad\text{i.e.,}\quad q_0=\tfrac{\varepsilon}{2}=0.05,\; q_1=1-\tfrac{\varepsilon}{2}=0.95.
\]

\smallskip
\noindent\textbf{Numeric examples (natural log):}
\begin{itemize}
  \item \emph{Correct, confident:} $\mathbf{p}=[0.10,\,0.90],\; y{=}1$
  \[
  \mathcal{L}_{\text{CE}}=-\ln(0.90)=\mathbf{0.1054},\qquad
  \mathcal{L}_{\text{CE+LS}}= -\big(0.05\ln 0.10 + 0.95\ln 0.90\big)=\mathbf{0.2152}.
  \]
  \item \emph{Correct, unsure:} $\mathbf{p}=[0.40,\,0.60],\; y{=}1$
  \[
  \mathcal{L}_{\text{CE}}=-\ln(0.60)=\mathbf{0.5108},\qquad
  \mathcal{L}_{\text{CE+LS}}= -\big(0.05\ln 0.40 + 0.95\ln 0.60\big)=\mathbf{0.5350}.
  \]
  \item \emph{Wrong, overconfident:} $\mathbf{p}=[0.90,\,0.10],\; y{=}1$
  \[
  \mathcal{L}_{\text{CE}}=-\ln(0.10)=\mathbf{2.3026},\qquad
  \mathcal{L}_{\text{CE+LS}}= -\big(0.05\ln 0.90 + 0.95\ln 0.10\big)=\mathbf{2.1927}.
  \]
\end{itemize}

\noindent\textbf{Takeaway.} Label smoothing slightly \emph{increases} loss for very confident correct predictions (discourages overconfidence), barely changes moderately confident ones, and \emph{reduces} the penalty for overconfident mistakes. This calibrates probabilities and improves generalization.

\subsection{Use of \texttt{tanh} Activations in Convolutional Layers}
While ReLU is the common default in deep CNNs, for oscillatory audio signals (waveforms) it discards all negative values. This can be problematic in tasks such as acoustic classification, where both positive and negative waveform amplitudes carry important information about phase and resonance. Using \texttt{tanh} instead preserves the sign of activations, while still introducing nonlinearity and bounding outputs.

\paragraph{Definition.}
The $\tanh$ activation is
\[
\tanh(x) \;=\; \frac{e^x - e^{-x}}{e^x + e^{-x}},
\]
with range $(-1,1)$. It is symmetric around zero and saturates for large $|x|$.

\paragraph{Why \texttt{tanh} helps.}
\begin{itemize}
  \item \textbf{Preserves polarity:} Positive and negative amplitudes are both kept, unlike ReLU which clamps all negatives to $0$.
  \item \textbf{Better fit for waveforms:} Acoustic signals are naturally centered around zero; forcing non-negative activations removes half the information.
  \item \textbf{Smoothing effect:} The saturation in $(-1,1)$ prevents extreme outliers from dominating the representation.
\end{itemize}

\paragraph{Numeric illustration.}
Consider three sample convolution outputs:
\[
x = [-2.0,\;-0.5,\;0.5,\;2.0].
\]

\begin{itemize}
  \item With ReLU:
  \[
  f(x) = [0,\;0,\;0.5,\;2.0],
  \]
  negative values vanish entirely.
  \item With tanh:
  \[
  f(x) = [-0.964,\;-0.462,\;0.462,\;0.964],
  \]
  negative amplitudes are retained and mapped symmetrically.
\end{itemize}

\paragraph{Effect in our models.}
By preserving sign information, \texttt{tanh} activations allowed the CNN layers to exploit both compression (positive) and rarefaction (negative) parts of the waveform. In practice (Phase~2 experiments), inserting \texttt{tanh} after convolutional layers improved calibration and added roughly $+2\%$ accuracy relative to ReLU-based baselines.

\subsection{Binary classification with two output neurons: test-time prediction and training-time loss}

\paragraph{Setup.}
The model ends with two outputs (logits) for each input, one per class:
\[
\mathrm{logits} = [z_0,\, z_1],\qquad z_i\in\mathbb{R}.
\]
These are unnormalized scores (not probabilities).

\paragraph{Training-time loss (CrossEntropyLoss).}
With target \(y\in\{0,1\}\), \texttt{CrossEntropyLoss} computes the negative log-likelihood of the true class. Writing softmax explicitly,
\[
p_i=\frac{e^{z_i}}{e^{z_0}+e^{z_1}},\qquad
\mathcal{L}=-\log p_y.
\]
For numerical stability, PyTorch uses the log-sum-exp identity (without explicitly forming softmax):
\[
\boxed{\;\mathcal{L} \;=\; -\,z_y \;+\; \log\!\big(e^{z_0}+e^{z_1}\big)\;}
\]

\paragraph{Worked numerical examples (training).}
\textbf{Example A (confident and correct).} Let \([z_0,z_1]=[2.1,-1.3]\) and \(y=0\).
\[
\begin{aligned}
e^{z_0}&=8.1662, \quad e^{z_1}=0.2725,\\
e^{z_0}+e^{z_1}&=8.4387,\\
p_0&=\tfrac{8.1662}{8.4387}=0.9677,\quad p_1=0.0323,\\
\mathcal{L}&=-\log(0.9677)=0.0328.
\end{aligned}
\]
Stable form:
\[
\mathcal{L}=-2.1+\log(8.4387)=0.0328.
\]

\noindent
\textbf{Example B (confident and wrong).} Same logits but \(y=1\).
\[
p_1=0.0323,\quad \mathcal{L}=-\log(0.0323)=3.4328.
\]

\noindent
\textbf{Example C (uninformative).} \([z_0,z_1]=[0,0]\). Then
\[
p_0=p_1=\tfrac{1}{2},\quad \mathcal{L}=-\log\tfrac{1}{2}=\log 2=0.6931.
\]

\noindent
\textbf{Example D (mild margin).} \([z_0,z_1]=[-0.2,0.2]\).
\[
p_0=0.4013,\; p_1=0.5987,\;
\mathcal{L}(y{=}0)=0.9130,\;\mathcal{L}(y{=}1)=0.5130.
\]

\paragraph{Test-time prediction.}
At inference, PyTorch predicts the class index with the largest logit:
\[
\hat{y}=\arg\max\{z_0, z_1\}.
\]
This is equivalent to taking \(\arg\max\) over softmax probabilities, because softmax preserves ordering.

\paragraph{Worked numerical examples (test).}
\textbf{Example A (clear margin).} \([z_0,z_1]=[2.1,-1.3]\).
\[
\hat{y}=\arg\max\{2.1,-1.3\}=0.
\]
Probabilities: \(p_0=0.9677,\; p_1=0.0323\), same decision.

\noindent
\textbf{Example B (ambiguous case).} \([z_0,z_1]=[0.4,\,0.41]\).
\[
\hat{y}=\arg\max\{0.4,\,0.41\}=1.
\]
Probabilities: \(p_0=0.4975,\; p_1=0.5025\). Prediction: \(\hat{y}=1\).

\noindent
\textbf{Example C (tie).} \([z_0,z_1]=[0,0]\).
\[
p_0=p_1=0.5,\qquad \arg\max\{0,0\}=0 \text{ (PyTorch picks the first index in a tie).}
\]

\paragraph{Discussion.}
The Phase~2 Waveform ResNet reaches an accuracy of \textbf{0.88}, which is notably lower than the Phase~1 result (0.98). This drop reflects the increased difficulty of Phase~2: the input signals are much shorter, which reduces the amount of temporal context the model can learn from. Although the model is an end-to-end ResNet and therefore has the capacity to extract raw waveform features directly, the results suggest that it does not fully capture the discriminative patterns of the Tight class.

The confusion matrix in Figure~\ref{fig:phase2_resnet_confusion_article} highlights this imbalance. While Drummy samples are identified reliably (recall = 0.92), the model misclassifies a substantial portion of Tight roofs (recall = 0.83), often labeling them as Drummy. This indicates that the model tends to favor the majority class features (Drummy) while struggling to learn the tighter spectral or temporal signatures of Tight roofs. Precision for Tight is slightly better (0.89), but the overall F1 of 0.86 remains weaker than what might be expected from a balanced classifier.

In practice, this means the Phase~2 ResNet is more cautious about calling a roof Tight, preferring instead to err on the side of Drummy. While this reduces the risk of overlooking a hazardous roof, it also undermines the reliability of recognizing genuinely safe areas. Future iterations of the model may need stronger feature engineering or hybrid architectures (e.g., combining waveform and spectral representations) to improve recall on the Tight class and reduce the overall misclassification rate.

\paragraph{Thresholding and boundary separation.}
Unlike Phase~1, where the end-to-end ResNet produced scores that clustered cleanly away from the threshold (Drummy near 0.0, Tight near 1.0), the Phase~2 MFCC+ResNet shows many predictions lying closer to the decision boundary at 0.5 (Figures~\ref{fig:p2_scores_by_index}--\ref{fig:p2_score_distribution}). This raises concerns about how sharply the model separates the two classes. In particular, several Drummy samples receive scores above 0.4, while a number of Tight samples fall below 0.6, creating overlap near the threshold and increasing the risk of misclassification.

This behavior suggests that the model has learned useful discriminative features from the MFCCs, but the boundary between Drummy and Tight is less distinct than in Phase~1. Some of these ambiguous cases could be genuine “borderline” acoustics, where the impact sound shares characteristics of both classes, or potential mishits in data collection. Introducing an additional label such as \emph{Unknown} or \emph{Ambiguous} could help capture these uncertain regions, allowing the classifier to flag low-confidence predictions rather than forcing a binary decision. Such a strategy would be especially valuable in safety-critical applications, where acknowledging uncertainty is preferable to overconfident but incorrect predictions.

\section{Model Discussion}
\begin{itemize}
    \item In Phase 1, the classification task has been pushed to its limits — the end-to-end \textbf{ResNet} model achieved \(\textbf{98.38\%}\) accuracy, outperforming even MFCC-based approaches.
    \item Across Phase 2 experiments, the \textbf{MFCC + ResNet} model achieved the highest \textbf{F1-score} overall, followed closely by the \textbf{MFCC + MLP} model.
    \item In terms of class-level performance:
        \begin{itemize}
            \item \textbf{MFCC + ResNet} showed more balanced \textbf{recall} across both “drummy” and “tight” classes, indicating better discrimination.
            \item \textbf{MFCC + MLP} also performed well but had slightly less balanced recall, suggesting a small bias towards one class.
        \end{itemize}
    \item Incorporating \textbf{label smoothing} led to an accuracy increase of approximately \(\textbf{+2\%}\), likely due to reducing overconfidence and improving generalization.
    \item Adding a \textbf{tanh activation} after convolution layers contributed another \(\textbf{+2\%}\) accuracy gain.
        \begin{itemize}
            \item \textbf{Intuition:} \texttt{tanh} preserves negative values, which may store important information about waveform amplitudes — especially for oscillatory signals where polarity matters.
        \end{itemize}
\end{itemize}

\section{Future Work}
\begin{itemize}
    \item Address the challenge of having two distinct datasets (Phase 1 and Phase 2) with different input dimensions:
        \begin{itemize}
            \item Phase 1 recordings: 0.75s duration, 36{,}000 samples.
            \item Phase 2 recordings: 0.1s duration, 4{,}800 samples.
        \end{itemize}
        \begin{itemize}
            \item Current approach involves resampling one dataset to match the other, which may lead to loss of information or introduce artifacts.
            \item A more elegant solution is needed to handle both datasets without resampling.
        \end{itemize}
    \item Develop a unified model architecture that can \textbf{process both dataset types} directly, without needing to resample the data to match a single fixed input dimension.
    \item Implement a more \textbf{robust preprocessing} pipeline:
        \begin{itemize}
            \item Here, \textbf{robust} means capable of automatically detecting and removing \textbf{mishits} or invalid recordings before feature extraction.
            \item This would involve statistical anomaly detection or learned filtering like similarity search with manually identified mishits to ensure only valid impacts are classified.
        \end{itemize}
\end{itemize}

\section{Conclusion}
\begin{itemize}
    \item \textbf{PCA-based methods} in Phase 1 provided a good starting point, showing that linear feature extraction can capture some structure in the data. However, PCA could not fully model the \textbf{temporal and non-linear patterns} present in acoustic signals.  
    \item Moving towards \textbf{deeper learning architectures} (ResNet, MFCC+CNN, MLP hybrids) demonstrated clear gains in accuracy and class balance, confirming that raw waveforms and spectrogram-like features benefit from hierarchical feature learning.  
    \item Architectural refinements such as \textbf{tanh activations} and \textbf{label smoothing} helped reduce overconfidence and improved generalization, adding small but consistent boosts in performance.  
    \item Overall, results show that while PCA is useful for dimensionality reduction, \textbf{end-to-end deep learning pipelines} are far more effective for real-world classification of mine roof impacts. This underscores the importance of non-linear feature extraction in safety-critical acoustic applications.  
\end{itemize}
